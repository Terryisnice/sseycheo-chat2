# **스쳐챗(Sseucheo Chat) PRD (제품 요구사항 정의서)**

## **1\. 제품 개요 (Product Overview)**

* **제품명:** 스쳐챗 (Sseucheo Chat)  
* **플랫폼:** iOS / Android 웹뷰 또는 크로스플랫폼 (MVP는 웹 기반 권장)  
* **한 줄 요약:** 10m 이내 초근접 거리 기반 익명 채팅과 목적형 오프라인 셔플챗을 결합한 하이브리드 소셜 앱.  
* **핵심 가치:** 1\. 외모/스펙이 아닌 취향(설문)과 대화(텍스트) 중심의 매칭.  
  2\. 시공간의 제한(10m, 이벤트)이 주는 긴장감과 희소성 제공.  
  3\. '어장' 시스템을 통한 지속적인 관계 관리 체계 구축.

## **2\. 문제 정의 및 해결책 (Problem & Solution)**

* **Problem:** 기존 데이팅 앱의 외모 지상주의 피로도 및 근거리 앱의 '밀집도(Density)' 부족 문제.  
* **Solution:** 사진 없는 텍스트/설문 중심의 Vibe Filter 도입 및 오프라인 '셔플챗' 이벤트를 통한 초기 유저 밀집도 확보.

## **3\. 핵심 유저 플로우 (User Flow)**

### **Flow A. 셔플챗 (이벤트 기반 만남 \- MVP 핵심)**

1. **입장/로그인:** 소셜 로그인(카카오/구글/애플 등)을 통해 인증 후 닉네임 설정.  
2. **이벤트 체크인:** 관리자가 생성한 현장 이벤트 방에 접속(체크인).  
3. **릴레이 채팅:** 시스템이 10분 단위로 파트너를 자동 매칭하여 1:1 채팅방 생성.  
4. **종료 및 선택:** 라운드 종료 1분 전 경고 알림. 종료 시 상대를 '어장'에 추가할지 묻는 팝업 표시.  
5. **다음 라운드:** 선택 완료 후 다음 파트너와 새로운 채팅방 오픈 (총 4라운드 진행).  
6. **최종 매칭:** 모든 라운드 종료 후, 상호 '어장'에 추가한 유저끼리만 영구 채팅방 활성화.

## **4\. 기능 요구사항 (Functional Requirements)**

### **4.1. 회원가입 및 프로필 모듈**

* $$MVP$$  
* 소셜 로그인 도입: Firebase Authentication을 활용하여 소셜 로그인(Google, Kakao 등) 필수 적용.  
  * *이유: 1\) '어장' 데이터의 영구 보존(기기 변경/앱 삭제 대응), 2\) 악성 유저 차단을 위한 고유 식별자(UID) 확보.*  
* **최소 프로필:** 소셜 로그인 성공 후, 앱 내에서 사용할 '익명 닉네임'만 설정. (실명, 프로필 사진은 앱 내 노출 금지)

### **4.2. 채팅 모듈 (안전장치 포함)**

* **텍스트 전용:** 이미지, 동영상, 음성, 이모티콘 전송 제한 (오직 텍스트만 가능).  
* **타이머 연동:** 남은 대화 시간이 상단에 실시간으로 표시됨.  
* $$MVP$$  
* 신고 및 차단 기능: 채팅방 내 원터치 '신고' 버튼 제공. 신고 시 해당 유저는 블랙리스트에 등록되어 추가 매칭을 방지하고, 관리자가 Firebase DB를 통해 악성 계정을 정지(UID 차단)할 수 있는 기반 마련.

### **4.3. 셔플챗 이벤트 모듈 (관리자 권한 포함)**

* **비밀 관리자 모드:** 앱 내 특정 영역(예: 로고 5번 탭) 진입 시 관리자 모드 활성화 (특정 관리자 UID에만 권한 부여).  
* **현장 방 만들기:** 관리자가 즉석에서 이벤트명(예: "일산 웨돔 테스트"), 인원수(4:4), 라운드 시간(10분)을 설정하여 방 생성.  
* **타이머 & 셔플 로직:** \* 종료 **1분 전** 채팅방 내에 시스템 경고 메시지 붉은색으로 노출 ("곧 대화가 종료됩니다").  
  * **종료 시점:** 대화방이 즉시 잠기며, 화면 중앙에 \[이 사람을 어장에 추가하시겠습니까? (예/아니오)\] 팝업 강제 표시.  
  * 양측 모두 선택을 완료해야 다음 파트너와 매칭 진행.

### **4.4. 어장(Fishbowl) 시스템 모듈**

* **히스토리 저장:** 어장 추가 시 만남 날짜, 장소 자동 태그. (소셜 계정에 연동되어 안전하게 보관됨)  
* **비밀 메모 기능:** 각 사용자 카드에 텍스트 메모를 남길 수 있는 기능.

## **5\. UI/UX 및 디자인 시스템 (Design Vibe)**

* **디자인 테마:** **"가장 흔하고 평범한 스타일 (Stealth & Utility Mode)"**  
* **시각적 특징:** \* 데이팅 앱 특유의 핑크/퍼플 등 화려한 색상 철저히 배제.  
  * 카카오톡 기본 테마, 텔레그램, 혹은 스마트폰 기본 문자(메시지) 앱처럼 하얀 배경에 무채색(검정/회색) 텍스트 위주.  
  * 남들이 어깨너머로 봤을 때 '업무용 메신저'나 '흔한 유틸리티 앱'으로 착각할 정도로 심심하고 깔끔하게 디자인.  
  * 긴장감은 디자인이 아닌 '1분 전 경고 타이머'와 '어장 선택'이라는 기능적 UI에서만 발생하도록 유도.

## **6\. MVP (Phase 1\) 개발 범위 정의**

$$목표$$

안전하고 영속적인 4:4 셔플챗 진행을 위한 일산 테스트용 코어 로직 구현

* **포함 범위 (AI 코딩 시 지시사항):**  
  * **소셜 로그인 및 닉네임 설정:** Firebase Auth (구글 로그인 등) 연결.  
  * **관리자 기능:** 방 생성 및 타이머 시작 권한 (Firebase DB 사용, 특정 UID만 관리자 권한).  
  * **셔플 로직:** 10분 채팅 \-\> 1분 전 경고 \-\> 종료 후  
  * $$어장 추가$$  
  * 선택 팝업 \-\> 다음 매칭.  
  * **신고/차단 기반:** 채팅방 내 단순 신고 버튼 (클릭 시 DB에 로그 남김).  
  * **흔한 디자인:** 흰 배경, 시스템 폰트, 기본 말풍선 형태의 UI.  
* **제외 범위:**  
  * BLE 기반 10m 탐색 기능 (Phase 2로 연기).  
  * 결제 시스템 (현장 테스트이므로 MVP에서는 무료로 진행).
