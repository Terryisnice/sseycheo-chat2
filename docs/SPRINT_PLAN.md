# 스쳐챗 MVP 스프린트 계획

## 운영 원칙

- 기간: 주 단위 스프린트 (월~일)
- 우선순위: `P0 -> P1 -> P2`
- 완료 정의(DoD): 기능 구현 + 기본 테스트 통과 + 문서 반영
- 기술 스택: React, Tailwind CSS, Firebase(Auth/Firestore)

## Sprint 1 (1주차) - MVP 뼈대 구축

### 목표
- 모바일 웹 기본 구조와 Firebase 연동을 완료한다.
- 인증/권한/이벤트 생성까지 "시작 가능한 상태"를 만든다.

### 작업 항목
- [ ] React + Tailwind 프로젝트 구조 정리 및 공통 레이아웃 구축
- [ ] Firebase 초기 세팅(Auth/Firestore) 및 환경변수 정리
- [ ] Google 로그인 + 익명 닉네임 온보딩 구현
- [ ] 사용자 문서(`users/{uid}`) 저장 및 기본 권한 필드 반영
- [ ] 관리자 UID 기반 접근 제어 구현
- [ ] 관리자 이벤트 생성 화면 구현 (`/admin`)
- [ ] 이벤트 컬렉션(`events`) 기본 스키마 반영
- [ ] 체크인/대기열 등록 기본 플로우 구현

### 완료 기준
- [ ] 일반 사용자는 로그인 -> 닉네임 설정 -> 로비 진입이 가능하다.
- [ ] 관리자는 이벤트 방을 생성할 수 있다.
- [ ] Firestore에서 사용자/이벤트 문서가 정상 생성된다.

### 리스크 및 대응
- Firebase 권한 규칙 미비 시 데이터 노출 위험 -> 초기에 최소 권한 규칙부터 적용
- 모바일 화면 깨짐 가능성 -> 주요 해상도(iPhone/Android) 우선 점검

## Sprint 2 (2주차) - 셔플챗 코어 완성

### 목표
- 4라운드 셔플챗 핵심 플로우를 동작시킨다.
- 신고/차단 포함 최소 안전장치를 붙인다.

### 작업 항목
- [ ] 라운드 상태 전이 구현 (시작/진행/종료)
- [ ] 라운드별 1:1 매칭 및 채팅방 생성 로직 구현
- [ ] 텍스트 전용 채팅 UI 구현 (미디어 전송 제거)
- [ ] 라운드 타이머 + 종료 1분 전 경고 메시지 구현
- [ ] 종료 시 채팅 잠금 + 어장 선택 팝업 구현
- [ ] 양측 선택 완료 후 다음 라운드 매칭 진행 구현
- [ ] 4라운드 종료 후 상호 선택 유저 영구 채팅방 생성
- [ ] 신고 버튼 + `reports` 저장 + 재매칭 제외 로직 구현

### 완료 기준
- [ ] 로그인 유저 8명 기준 테스트 이벤트에서 4라운드가 끝까지 진행된다.
- [ ] 라운드 종료/전환 타이밍이 정상 동작한다.
- [ ] 신고된 사용자에 대해 재매칭 제외가 반영된다.

### 리스크 및 대응
- 실시간 동기화 오차 -> 서버시간 기준으로 타이머 보정
- 매칭 예외(이탈/미응답) -> 타임아웃 정책과 기본 fallback 정의

## Sprint 3 (3주차) - 어장/안정화/테스트

### 목표
- 어장 기능과 운영 안정화를 마무리하고 MVP 테스트 준비를 완료한다.

### 작업 항목
- [ ] 어장 저장(만남 날짜/이벤트 태그) 및 카드 UI 구현
- [ ] 어장 비밀 메모 CRUD 구현
- [ ] 관리자 차단 플로우 고도화 (`isBlocked=true`)
- [ ] 실패/이탈/재시도 UX 정리
- [ ] 핵심 E2E 시나리오 테스트 체크리스트 수행
- [ ] 운영 지표(체크인 수/완주율/신고율) 로그 점검

### 완료 기준
- [ ] 어장 목록과 메모가 사용자 단위로 정상 동작한다.
- [ ] 관리자 차단 후 접근/매칭 제한이 적용된다.
- [ ] 핵심 시나리오 테스트 통과율이 목표 수준에 도달한다.
